The Jumper widget wraps routing, approvals, gas estimation, and status tracking into a single UI component. Use it to launch production-ready bridging and swapping without rebuilding core flows.

<Info>
  **Prerequisites**
  - Node.js 18+
  - React 18 or Next.js 13+ (App Router or Pages)
  - An API key for production traffic
</Info>

## Install

```bash
npm install @lifi/widget
```

## Add the widget

```tsx
'use client';
import { LiFiWidget } from '@lifi/widget';

const theme = {
  primary: '#6D28D9',
  secondary: '#1E0F3F',
  borderRadius: 16
};

export function JumperWidget() {
  return (
    <div className="h-[720px] w-full max-w-3xl">
      <LiFiWidget
        integrator="jumper-docs"
        variant="expandable"
        appearance={theme}
        config={{
          disableTelemetry: false,
          bridges: { allow: ['hop', 'stargate', 'celer'] },
          exchanges: { allow: ['uniswap', '1inch', 'paraswap'] },
          insurance: true
        }}
      />
    </div>
  );
}
```

<Tip>Wrap the widget in a parent with a fixed height. The component is fully responsive inside that container.</Tip>

## Configure tokens and chains

Use the config props to restrict assets and networks for your business logic.

```tsx
<LiFiWidget
  integrator="jumper-docs"
  config={{
    fromChain: 1,
    toChain: 137,
    fromToken: '0xEeeeeEeeeEeEeeEeEeEeeEEEeeeeEeeeeeeeEEeE',
    toToken: '0x7ceB23fD6bC0adD59E62ac25578270cFf1b9f619',
    slippage: 0.005,
    fee: {
      integrator: 'jumper-docs',
      fee: 0.0015,
      feeRecipient: '0x0000000000000000000000000000000000000000'
    }
  }}
/>
```

- **Chain controls:** allow/deny lists for bridges and exchanges.
- **Fees:** set an integrator fee and destination address.
- **Appearance:** primary/secondary colors, radius, shadows, and font family.

## Events and callbacks

Listen to lifecycle events to keep your UI in sync.

```tsx
<LiFiWidget
  onRouteExecutionStarted={route => console.log('Executing', route.id)}
  onRouteExecutionFailed={(route, error) => console.error(route.id, error)}
  onRouteExecutionCompleted={route => console.log('Completed', route.id)}
/>
```

Use these callbacks to fire analytics, notify your backend, or update in-app notifications.

## Deploying to production

- Provide your `x-api-key` for production traffic.
- Cache token metadata and chain lists to reduce cold-start latency.
- Surface route warnings (paused bridge, low liquidity) before users confirm.
- Subscribe to webhooks or poll `/v1/status/{transferId}` to keep users informed if they close the tab.
