Get Jumper running in minutes. Pick the integration mode, drop in the widget or call the API, and ship a reliable cross-chain experience.

## Step 1: Choose your integration

<AccordionGroup>
  <Accordion icon="window-maximize" title="Widget (fastest)">
    Drop-in UI that handles routing, gas, approvals, and user flows. Recommended if you want a production-ready experience with minimal effort.
  </Accordion>
  <Accordion icon="code" title="API (full control)">
    Use the HTTP API to fetch quotes, build transactions, and manage settlement yourself. Recommended for custom UI or server-side flows.
  </Accordion>
</AccordionGroup>

## Step 2: Add the widget

Install the Jumper widget and render it inside your app.

```bash
npm install @lifi/widget react react-dom
```

```tsx
'use client';
import { LiFiWidget } from '@lifi/widget';

export function JumperWidget() {
  return (
    <div className="h-[720px] w-full max-w-3xl rounded-2xl border border-purple-200/50 shadow-lg">
      <LiFiWidget
        integrator="jumper-docs"
        variant="wide"
        appearance={{
          primary: '#6D28D9',
          secondary: '#1E0F3F'
        }}
      />
    </div>
  );
}
```

<Tip>Wrap the widget in a responsive container so it scales across mobile and desktop.</Tip>

## Step 3: Call the API

Use the API when you need to pre-validate routes, run compliance checks, or render your own UI.

```ts
async function getQuote() {
  const response = await fetch('https://api.jumper.exchange/v1/quote?fromChainId=1&toChainId=137&fromTokenAddress=0xEeeeeEeeeEeEeeEeEeEeeEEEeeeeEeeeeeeeEEeE&toTokenAddress=0x7ceB23fD6bC0adD59E62ac25578270cFf1b9f619&fromAmount=1000000000000000000&slippage=0.005', {
    headers: { 'x-api-key': process.env.NEXT_PUBLIC_JUMPER_API_KEY ?? '' }
  });

  if (!response.ok) throw new Error('Failed to fetch quote');
  return response.json();
}
```

With a route in hand, submit the transaction payload returned by `/v1/transfer` using your signer or server-side relayer.

## Optimize for production

- **Add monitoring:** Track success, reverts, and bridge/DEX status. Subscribe to the status webhook or poll `/v1/status/{transferId}`.
- **Control risk:** Set max slippage per chain, enforce allowlists/denylists, and cap daily volume.
- **Handle approvals:** The widget manages approvals automatically. For custom UIs, request approvals before broadcasting the transfer transaction.
- **Surface fees:** Show gas estimates and protocol fees from the quote to avoid user surprise.

## Security and compliance

- Use your own `integrator` tag to segment traffic and manage rate limits.
- Keep API keys in server-side environments or protected client env vars.
- Validate destination addresses and amount bounds before sending the transaction payload to the chain.
- Monitor [`status.li.fi`](https://status.li.fi) for bridge or DEX incidents.
