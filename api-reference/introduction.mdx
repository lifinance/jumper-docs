## Base URL

```
https://api.jumper.exchange
```

## Authentication

Send your API key with every request.

```http
x-api-key: YOUR_API_KEY
```

Use distinct keys per environment and rotate them regularly. Requests without a key are rate limited.

## Core workflow

1. **Quote:** call `/v1/quote` with source/destination chain, token, and amount.
2. **Transfer:** pass the `routeId` to `/v1/transfer` to receive a transaction payload.
3. **Broadcast:** send the transaction with your signer or relayer.
4. **Track:** poll `/v1/status/{transferId}` or subscribe to the status webhook to keep users informed.

## Rate limits

- Quotes: optimized for real-time UX; cache identical requests where possible.
- Transfers: signed requests are enforced; replay protection is applied to route IDs.
- Status: prefer webhooks; poll with exponential backoff if webhooks are unavailable.

## Errors

Errors follow a consistent structure:

```json
{
  "error": "ROUTE_UNAVAILABLE",
  "message": "No route found within your constraints"
}
```

Handle `429` by backing off and reusing cached quotes. For `400` errors, surface the message to users with remediation guidance (e.g., widen slippage or pick another bridge).
